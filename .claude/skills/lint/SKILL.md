# 代码检查

运行 ESLint 检查代码质量问题。

## 可用命令

- `pnpm lint` - 检查所有包

## 默认操作

运行 lint 检查，如果发现问题则修复：

```bash
pnpm lint
```

## 常见问题类型

- 未使用的变量/导入
- 缺少类型注解
- 代码风格不一致
- 潜在的 bug（如 == 而非 ===）

## 修复策略

1. 自动修复：大部分风格问题可以自动修复
2. 手动修复：类型错误、逻辑问题需要手动处理
3. 忽略规则：确实需要时可以添加 `// eslint-disable-next-line`

## 注意事项

- 不要盲目禁用规则，先理解为什么触发
- 保持代码风格一致性
