# 代码审查

对代码变更进行全面审查，检查质量、安全性和性能问题。

## 审查范围

使用 `git diff` 或 `git diff --staged` 获取变更内容进行审查。

## 审查维度

### 1. 代码质量
- 代码是否清晰易读
- 命名是否有意义
- 是否有重复代码
- 函数是否过长/复杂
- 是否遵循项目风格

### 2. 类型安全
- TypeScript 类型是否正确
- 是否避免使用 `any`
- 是否处理了 null/undefined

### 3. 安全性
- 是否有 XSS 风险（dangerouslySetInnerHTML）
- 是否有敏感信息泄露（API keys, secrets）
- 是否有 SQL/命令注入风险
- 用户输入是否验证

### 4. 性能
- 是否有不必要的重渲染
- 是否正确使用 useMemo/useCallback
- 是否有内存泄漏风险
- 大列表是否考虑虚拟化

### 5. 错误处理
- 异步操作是否有错误处理
- 边界情况是否考虑
- 用户体验是否友好

## 输出格式

```markdown
## 代码审查报告

### 发现的问题

#### 严重 (必须修复)
- [文件:行号] 问题描述

#### 建议 (推荐修复)
- [文件:行号] 问题描述

### 优点
- 列出代码中做得好的地方

### 总结
整体评价和建议
```

## 审查原则

- 保持建设性，提供具体建议
- 区分必须修复和可选优化
- 认可好的实践
